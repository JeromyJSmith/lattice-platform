# Local Harness Golden Path

This harness layer keeps its detailed canonical execution path in:

- `golden_path.md`

That file remains the detailed operational source because existing repo
references already point to it.

At the fractal scaffold level, this file exists so the harness directory
exposes the same minimum contract shape as the outer wrapper layers:

- `AGENTS.md`
- `CLAUDE.md`
- `MEMORY.md`
- `GOAL.md`
- `GOLDENPATH.md`

## Contract

For detailed execution steps, capability promotion paths, and harness proof
rules, read `golden_path.md`.

For harness catalog/config bindings, read `library.yaml` and
`agentics-library.md`.
